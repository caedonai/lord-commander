## Installation Options

Lord Commander offers different installation methods depending on your needs:

### 1. Library Mode (SDK Integration)

Install the CLI SDK for use in existing projects:

<Tabs>
  <Tab title="npm">
    ```bash
    npm install @lord-commander/cli-core
    ```
  </Tab>
  <Tab title="pnpm">
    ```bash
    pnpm add @lord-commander/cli-core
    ```
  </Tab>
  <Tab title="yarn">
    ```bash
    yarn add @lord-commander/cli-core
    ```
  </Tab>
</Tabs>

### 2. CLI Application Setup

Use the `init` command to create complete CLI applications:

<Tabs>
  <Tab title="CLI-Only">
    ```bash
    # Install CLI globally first
    npm install -g lord-commander
    
    # Initialize CLI-only project
    lord-commander init --quick --type=cli-only
    ```
  </Tab>
  <Tab title="CLI + API">
    ```bash
    lord-commander init --quick --type=cli-api
    ```
  </Tab>
  <Tab title="Full-Stack">
    ```bash
    lord-commander init --quick --type=full-stack
    ```
  </Tab>
</Tabs>

## Quick Setup with Init Command

The easiest way to get started is using the `init` command:

### Interactive Setup

```bash
lord-commander init
```

Follow the prompts to choose your setup type and configuration.

### Quick Setup Examples

```bash
# Library mode - just the SDK
lord-commander init --quick --type=library

# Library mode in project folder
lord-commander init --quick --type=library --output my-sdk

# CLI-only application (recommended approach)
lord-commander init --quick --type=cli-only --output my-cli --pm=pnpm

# Full-stack with dashboard in project folder (recommended)
lord-commander init --quick --type=full-stack --output my-project --pm=pnpm

# Full-stack with custom paths
lord-commander init --quick --type=full-stack --dashboard-path ./admin --api-path ./backend
```

## Verification

### Check Installation

Verify that Lord Commander is installed correctly:

```bash
lord-commander --version
lord-commander status
```

You should see output showing the CLI SDK version and system information.

### Test Basic Functionality

Run the help command to ensure everything is working:

```bash
lord-commander --help
```

This will display the available commands and options.

## Installation Flags

The `init` command supports several flags for customization:

| Flag               | Description                               | Values                                         | Default    |
| ------------------ | ----------------------------------------- | ---------------------------------------------- | ---------- |
| `--type`           | Setup type                                | `library`, `cli-only`, `cli-api`, `full-stack` | `cli-only` |
| `--quick`          | Skip interactive mode                     | `true`, `false`                                | `false`    |
| `--global`         | Install globally                          | `true`, `false`                                | `false`    |
| `--pm`             | Package manager                           | `npm`, `pnpm`, `yarn`                          | `npm`      |
| `--output`         | Output directory (creates project folder) | `string`                                       | -          |
| `--dashboard-path` | Custom path for dashboard-ui component    | `string`                                       | -          |
| `--api-path`       | Custom path for API component             | `string`                                       | -          |

### Examples with Flags

```bash
# Library mode with pnpm, installed globally
lord-commander init --quick --type=library --pm=pnpm --global

# CLI-only with yarn in project folder (recommended)
lord-commander init --quick --type=cli-only --output my-cli --pm=yarn

# Full-stack in project folder with pnpm (recommended)
lord-commander init --quick --type=full-stack --output my-project --pm=pnpm

# Full-stack with custom component paths
lord-commander init --quick --type=full-stack --dashboard-path ./admin --api-path ./server

# CLI+API in current directory
lord-commander init --quick --type=cli-api --pm=pnpm

# Interactive setup with project structure options
lord-commander init --type=full-stack
```

## Environment Setup

### Shell Completion (Optional)

Enable shell completion for a better CLI experience:

<Tabs>
  <Tab title="bash">
    ```bash
    # Add to ~/.bashrc
    eval "$(lord-commander completion bash)"
    ```
  </Tab>
  <Tab title="zsh">
    ```bash
    # Add to ~/.zshrc
    eval "$(lord-commander completion zsh)"
    ```
  </Tab>
  <Tab title="fish">
    ```bash
    # Add to ~/.config/fish/config.fish
    lord-commander completion fish | source
    ```
  </Tab>
</Tabs>

### Configuration Directory

Lord Commander creates a configuration directory at:

- **macOS/Linux:** `~/.config/lord-commander/`
- **Windows:** `%APPDATA%\lord-commander\`

This directory stores:

- User preferences
- Template cache
- Plugin configurations

## Troubleshooting

<AccordionGroup>
  <Accordion icon="package" title="Package Installation Failed">
    If `init` command fails to install packages:
    
    **Check internet connection and try different package manager:**
    ```bash
    lord-commander init --quick --type=library --pm=pnpm
    ```
    
    **Verify npm registry access:**
    ```bash
    npm config get registry
    npm ping
    ```
  </Accordion>

{" "}

<Accordion icon="triangle-exclamation" title="Permission Errors">
  If you encounter permission errors during global installation: **Use local
  installation instead:** ```bash lord-commander init --quick --type=library #
  (Installs locally, no global permissions needed) ``` **Or configure npm
  properly (better approach):** ```bash mkdir ~/.npm-global npm config set
  prefix '~/.npm-global' echo 'export PATH=~/.npm-global/bin:$PATH' >> ~/.bashrc
  source ~/.bashrc ```
</Accordion>

{" "}

<Accordion icon="network-wired" title="Network Issues">
  If installation fails due to network issues: **Try a different registry:**
  ```bash lord-commander init --quick --type=library --pm=npm --registry
  https://registry.npmjs.org/ ``` **Or configure npm proxy (if behind corporate
  firewall):** ```bash npm config set proxy http://proxy.company.com:8080 npm
  config set https-proxy http://proxy.company.com:8080 ```
</Accordion>

  <Accordion icon="exclamation-circle" title="Command Not Found">
    If `lord-commander` is not recognized:
    
    1. **Install the CLI globally first:**
    ```bash
    npm install -g lord-commander
    ```
    
    2. **Check your PATH and restart terminal**
    
    3. **Use npx as alternative:**
    ```bash
    npx lord-commander init
    ```
  </Accordion>
</AccordionGroup>
