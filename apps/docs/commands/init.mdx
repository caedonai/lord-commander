## Overview

The `init` command bootstraps new projects with intelligent defaults, modern tooling configurations, and industry best practices. It provides interactive prompts to customize your project setup based on your specific needs.

## Basic Usage

```bash
# Interactive initialization
lord-commander init

# Initialize with specific framework
lord-commander init --framework react

# Initialize with all options
lord-commander init --framework vue --language typescript --package-manager pnpm
```

## Command Options

### Framework Selection

| Option        | Description       | Supported Values                            |
| ------------- | ----------------- | ------------------------------------------- |
| `--framework` | Target framework  | `react`, `vue`, `angular`, `svelte`, `node` |
| `--version`   | Framework version | Auto-detected or specified                  |

```bash
# React with latest version
lord-commander init --framework react

# Vue 3 with Composition API
lord-commander init --framework vue --version 3

# Angular with standalone components
lord-commander init --framework angular --standalone
```

### Language Configuration

| Option       | Description            | Default      |
| ------------ | ---------------------- | ------------ |
| `--language` | Primary language       | `typescript` |
| `--strict`   | TypeScript strict mode | `true`       |
| `--jsx`      | JSX transform          | `react-jsx`  |

```bash
# JavaScript project
lord-commander init --language javascript

# TypeScript with strict mode
lord-commander init --language typescript --strict

# Custom JSX transform
lord-commander init --framework react --jsx react-jsx
```

### Package Management

| Option              | Description     | Auto-detected                  |
| ------------------- | --------------- | ------------------------------ |
| `--package-manager` | Package manager | From lockfile or global config |
| `--registry`        | npm registry    | `https://registry.npmjs.org/`  |

```bash
# Use pnpm
lord-commander init --package-manager pnpm

# Custom registry
lord-commander init --registry https://npm.company.com/
```

### Project Structure

| Option         | Description      | Default    |
| -------------- | ---------------- | ---------- |
| `--structure`  | Directory layout | `standard` |
| `--src-dir`    | Source directory | `src`      |
| `--public-dir` | Public assets    | `public`   |

```bash
# Custom structure
lord-commander init --structure monorepo

# Custom directories
lord-commander init --src-dir source --public-dir assets
```

## Advanced Options

### Monorepo Setup

```bash
# Initialize monorepo structure
lord-commander init --workspace monorepo --name my-workspace

# Add multiple apps
lord-commander init --workspace-app frontend --framework react
lord-commander init --workspace-app api --framework node
lord-commander init --workspace-lib shared-utils
```

### Custom Templates

```bash
# Use custom template
lord-commander init --template ./my-templates/custom-react

# Template from URL
lord-commander init --template https://github.com/user/my-template

# Template from npm package
lord-commander init --template @company/project-template
```

### Environment Configuration

```bash
# Initialize with environment configs
lord-commander init --environments development,staging,production

# Docker configuration
lord-commander init --docker --docker-compose

# CI/CD templates
lord-commander init --ci github-actions --cd vercel
```
